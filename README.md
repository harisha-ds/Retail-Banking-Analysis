# Retail Banking Customer & Transaction Analytics

## Project Overview
This project analyzes retail banking data to understand **customer behavior, transaction activity, and branch performance**. An interactive dashboard was built using **Power BI** to help stakeholders monitor key metrics such as revenue, customer growth, and transaction trends.

The goal of this project is to convert raw banking data into **actionable insights that support better business decisions**.

---

## Business Problem
Retail banks generate large amounts of customer and transaction data, but without proper analytics it becomes difficult to monitor performance.

Management faced several challenges:

- Uneven transaction growth across branches  
- Increasing number of inactive accounts  
- Limited visibility into customer behavior  
- Revenue dependency on a small group of customers  

This project aims to solve these issues by building a **centralized analytics dashboard**.

---

## Tools Used

- **Power BI** – Dashboard development and data visualization  
- **DAX** – KPI calculations and business metrics  
- **Power Query** – Data transformation and cleaning  

---

## Dataset Overview

The dataset contains **8,500+ banking records** across multiple tables.

Main tables include:

- **Customers** – Customer demographics and tenure  
- **Accounts** – Account types and status  
- **Transactions** – Financial transactions and amounts  
- **Branches** – Branch location details  

---

## Data Model

A **star schema data model** was implemented.

### Fact Table
- Transactions  

### Dimension Tables
- Customers  
- Accounts  
- Branches  
- Date  

This structure enables efficient analysis across different business dimensions such as customers, branches, and time.

---

## Dashboard Pages

### 1. Executive Summary
Provides a high-level overview of banking performance using KPIs and trend analysis.

**KPIs included:**

- Total Customers  
- Total Accounts  
- Total Transactions  
- Total Revenue  
- Average Transaction Value  

Visualizations show revenue and transaction trends over time.

---

### 2. Customer Insights
Analyzes customer demographics and distribution.

Key insights include:

- Customer distribution by city  
- Age group analysis  
- Customer acquisition trends  
- Identification of high-value customers  

---

### 3. Account & Transaction Analysis
Examines banking product usage and transaction patterns.

Insights include:

- Account type distribution  
- Inactive account analysis  
- Transaction type breakdown  
- Identification of high-value transactions  

---

### 4. Branch Performance
Evaluates operational performance across different branches.

Insights include:

- Revenue generated by each branch  
- Transaction activity by branch  
- Comparison of branch performance across cities  

---

## Key Insights

- A small group of **high-value customers contributes a significant portion of transaction value**.  
- **Savings accounts dominate the account portfolio**, reflecting a retail-focused customer base.  
- Some branches generate **higher revenue and transaction activity**, indicating differences in operational performance.  
- A portion of accounts remain **inactive**, highlighting opportunities for customer re-engagement.

---

## Business Recommendations

- Launch **targeted campaigns** to reactivate inactive accounts.  
- Develop **premium banking services** for high-value customers.  
- Strengthen support for **underperforming branches**.  
- Improve **customer acquisition strategies** in high-growth cities.

---

## Conclusion

This project demonstrates how business intelligence tools like **Power BI** can transform raw banking data into actionable insights. The dashboard enables decision-makers to monitor performance, understand customer behavior, and identify growth opportunities.
